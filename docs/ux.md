# UX-концепция

## Персоны

### Персона 1: Студент с несколькими подписками

- Цели: быстро понять, сколько денег уходит в месяц, и не забыть об оплате.
- Важно в первые 30 сек: общая сумма расходов, ближайшие списания, активные подписки.

### Персона 2: Пользователь, оптимизирующий бюджет

- Цели: найти дорогие или неиспользуемые подписки, отключить лишнее, контролировать лимиты по категориям.
- Важно в первые 30 сек: превышения лимитов, сортировка по стоимости, статус активных подписок.

## User Stories

1. Как пользователь, хочу войти в аккаунт, чтобы видеть только свои подписки.
   - Шаги: открыть login, ввести email и пароль, перейти на dashboard.
   - Done-критерий: защищенные страницы недоступны без авторизации.

2. Как пользователь, хочу добавить подписку, чтобы она попала в общий расчет расходов.
   - Шаги: открыть подписки, заполнить форму, выбрать категорию, сохранить.
   - Done-критерий: подписка появляется в списке и влияет на месячную сумму.

3. Как пользователь, хочу фильтровать и сортировать подписки, чтобы найти расходы для оптимизации.
   - Шаги: ввести запрос, выбрать категорию/статус, сменить сортировку.
   - Done-критерий: список обновляется без перезагрузки страницы.

4. Как пользователь, хочу видеть лимиты категорий, чтобы понимать превышение бюджета.
   - Шаги: открыть dashboard, сравнить расходы и лимиты.
   - Done-критерий: категории с превышением выделены визуально.

## Дизайн-система (Taiga UI 4)

- Используемые компоненты: `TuiRoot`, `TuiButton`, `TuiLoader`, `TuiNotification`, `TuiBadge`, `TuiProgressBar`.
- Кастомные компоненты: shell layout, dashboard cards, subscription form, subscription list.
- Визуальный стиль: светлый рабочий интерфейс, нейтральный фон, зеленый основной акцент, отдельные цвета категорий.
- Адаптивность: desktop layout с формой рядом со списком, mobile layout в одну колонку.

## Приоритеты

- Первый экран: месячные расходы, количество активных подписок, ближайшие платежи, превышения лимитов.
- Вторично: CRUD-форма, фильтры, сортировка, управление статусом.
- Сознательно убрано: реальные платежи, email/push-уведомления, OAuth, импорт банковских операций.
